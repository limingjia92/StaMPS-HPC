## Overview

This directory contains the high-performance C/C++ core binaries for the **StaMPS-HPC** framework. These modules are responsible for the initial ingestion and selection of Persistent Scatterer (PS) candidates from interferograms generated by InSAR processors (e.g., **GAMMA**, **ISCE**, or **GMTSAR**).

We have refactored the legacy code to leverage **OpenMP parallelization**, **optimized block I/O**, and **modern CPU instruction sets**, significantly reducing the processing time for large-scale datasets.

---

## 1. Amplitude Calibration (`calamp`)

**Function:** Calculate amplitude calibration constant for SLC files.

**Optimization Strategy:**
* **Parallelization:** Refactored the core processing loop using **OpenMP** to enable multi-threaded execution.
* **I/O Architecture:** Transitioned from legacy line-by-line reading to a **Full-Memory / Block I/O** strategy. This minimizes disk system calls by loading large chunks of `.rslc` data directly into RAM.
* **Latency Reduction:** Significantly reduced CPU wait time associated with disk I/O.

**Performance Impact:**
* **Speedup:** ~70% reduction in execution time.
* **Recommendation:** Due to the I/O-bound nature of this module, using **NVMe SSDs** is highly recommended to maximize throughput.

**Compilation & Usage:**
```bash
# Compilation (Requires OpenMP)
g++ -o calamp calamp_opt.c -O3 -fopenmp -march=native

# Execution Example (with 4 threads)
export OMP_NUM_THREADS=4 
./calamp calamp.in 67546 calamp.out s 1 > calamp.log

```
## 2. PS/SB Selection (`selpsc_patch` & `selsbc_patch`)

**Function:** Selects PS candidates for PS/SB datasets within patches.

**Optimization Strategy:**
* **Buffered Reading:** Implemented a `LineInBufferMax` mechanism to handle data in optimized memory blocks rather than continuous small streams.
* **Memory Efficiency:** Optimized internal data structure allocation to better handle large-scale datasets without stack overflow.

**Performance Impact:**
* **Speedup:** ~40% reduction in execution time.
* **Note:** High I/O saturation. Running multiple concurrent instances on a single HDD may lead to disk contention; SSDs are preferred for concurrent patch processing.

**Compilation & Usage:**
```bash
g++ -O3 -march=native -ffast-math -o selpsc_patch selpsc_patch_opt.c -lm
g++ -O3 -march=native -ffast-math -o selsbc_patch selsbc_patch_opt.c -lm

```
## 3. Phase Stability Estimation (`pscphase`)

**Function:** Extract phase for PS candidates from complex interferograms.

**Optimization Strategy:**
* **I/O Logic:** Modified read logic to prioritize sequential reading over random seeking.
* **Seek Reduction:** Increased the file system input volume buffer to reduce mechanical disk head movement (seek latency).
* **Tunable Parameter:** Introduced `READ_VS_SEEK_THRESHOLD` (Default: 2MB). The system automatically decides whether to "read through" data gaps or "seek" to the next position.

**Performance Impact:**
* **Benefit:** Most effective for high-density PS point sets. While the raw speedup is marginal on modern SSDs, it significantly improves efficiency on enterprise-level HDDs.

**Compilation & Usage:**
```bash
g++ -O3 -o pscphase pscphase_opt.c

```
## 4. DEM and Coordinates Search (`pscdem` & `psclonlat`)

**Function:** Extract DEM height and geographic coordinates (Lon/Lat) for PS Candidates.

**Optimization Strategy:**
* **Algorithm Refactoring:** Updated search algorithms to improve lookup efficiency during coordinate assignment.
* **Instruction Optimization:** Applied `-march=native` to leverage SIMD and other modern CPU instructions.

**Performance Impact:**
* **Result:** Provides a cleaner execution profile. Absolute time reduction is minor as these modules are not the primary bottleneck.

**Compilation & Usage:**
```bash
g++ -O3 -march=native -o pscdem pscdem_opt.c
g++ -O3 -march=native -o psclonlat psclonlat_opt.c

```
